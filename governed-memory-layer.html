Governed Memory Layer

A gated append-only memory pipeline for multi-agent shared state.

The system treats agent notes as proposed writes, not canonical truth. Notes pass through interpretation, resolution, validation, and projection before they can alter shared state.

The useful part is the ambiguity boundary. If a note cannot be safely attached to an existing entity, it becomes pending state instead of silently corrupting memory.

01 Append-only event ledger for durable memory writes.

02 Canonical projected tables for plans, issues, constraints, decisions, results, task state, and learnings.

03 Pending queue for unresolved lifecycle updates.

04 Proof: GitHub repo / benchmark harness in repo.
